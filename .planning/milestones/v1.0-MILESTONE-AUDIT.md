## Integration Findings

### WARNING: Private workstream remote references not re-validated after merge (INT-01)

**Affects:** CFG-07  
`mergePrivateConfig` runs after `buildAndValidate` completes. `validateWorkstreams` checks workstream→remote name references only against the main-config remote set. A private `.git/.gitw` introducing both a new remote and a workstream that references that remote name will load without a cross-file referential integrity error.

**Action needed:** Add a post-merge validation pass or re-run `validateWorkstreams` after `mergePrivateConfig`.

### INFO: `sync_pair` from/to not validated against declared remote names (INT-02)

**Affects:** CFG-05  
`validateSyncPairFields` checks empty fields and duplicates but does not call `cfg.RemoteByName()` for `from`/`to` values. A sync_pair referencing a nonexistent remote name passes validation silently.

### INFO: Alias path warnings dropped on alias validation error (INT-03)

**Affects:** CFG-02, CFG-03  
Warnings accumulated by `warnNonConformingRepoPaths` are attached to a `cfg` that is discarded when `validateAliasFields` returns an error later in `buildAndValidate`. No runtime impact for valid configs.

### INFO: SpecFramework interface methods have no call sites (INT-04)

**Affects:** CFG-11  
`SpecFramework` interface methods (`PlanningDirExists`, `InitInstructions`, `ProhibitedActions`, `WorkspaceCreationProhibited`) are implemented but never called. The registry is validated at config load time but the resolved frameworks are not consumed through the interface. Expected for M1 — consumer call sites are future milestone work (M9: Agent Context Layer).
