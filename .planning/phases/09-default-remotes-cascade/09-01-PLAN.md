<objective>
Fix the `MergeWorkstream` and `MergeRepo` guard conditions in `pkg/config/config.go` so that an explicit empty slice (`[]string{}`) is treated as a deliberate "no remotes" override rather than being ignored. This is a prerequisite for the cascade resolver in Plan 02 to correctly implement the `nil` = fall through / `[]string{}` = stop cascade semantics (D-06, D-07, D-08, D-09 from CONTEXT.md).

Purpose: Without this fix, a repo or workstream config that sets `remotes = []` would be silently ignored by the merge logic, and the cascade would incorrectly fall through to the outer level.
Output: Two guard changes in `config.go`, updated test cases in `config_test.go` that verify the empty-slice behavior is now correctly treated as a replace (not ignored).
</objective>
